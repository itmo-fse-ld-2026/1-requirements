# Лабораторная работа №1
[![View on GitHub Pages](https://img.shields.io/badge/GitHub%20Pages-View%20PDF-blue?logo=github)](https://itmo-fse-ld-2026.github.io/1-requirements/)

Составить список требований, предъявляемых к разрабатываемому веб-сайту:

> Дельфийский оракул — всевозможные онлайн гадания - [https://magya-online.ru/](https://magya-online.ru/)

Требования должны делиться на следующие категории:

- **Функциональные**
  - Требования пользователей сайта
  - Требования владельцев сайта
- **Нефункциональные**

Требования необходимо оформить в соответствии с шаблонами RUP _(документ SRS - Software Requirements Specification)_. Для каждого из требований нужно указать его атрибуты _(в соответствии с методологией RUP)_, а также оценить и аргументировать приблизительное количество часов, требующихся на реализацию этого требования.

Для функциональных требований нужно составить UML UseCase-диаграммы, описывающие реализующие их прецеденты использования.

## Разработка

Для обеспечения единообразия окружения при компиляции документации рекомендуется использовать **Docker**. Это позволяет избежать проблем с отсутствующими шрифтами или разными версиями пакетов TeX.

### Сборка через Docker

Используйте следующую команду для запуска сборки внутри контейнера:

```bash
docker run --rm -v .:/data ghcr.io/itmo-fse-ld-2026/1-xetex-compiler:dev make pdf
```

**Примечание:** При запуске через Docker текущая директория монтируется в `/data`, что позволяет сохранять результат компиляции на вашей локальной машине.

> **Выбор тега образа:**
> * `:latest` — стабильная версия из ветки `main`.
> * `:dev` — актуальная версия для разработки из ветки `dev`.
> * `:pr-<номер>` — временные образы для проверки изменений в Pull Request (если был изменен Dockerfile).

### Команды Makefile

В проекте используется `Makefile` для автоматизации рутинных задач. Основные цели:

* **clean**: удаляет временные файлы компиляции и готовые PDF-файлы
* **pdf**: выполняет двойную компиляцию исходного кода через `xelatex` для корректного отображения ссылок и оглавления
